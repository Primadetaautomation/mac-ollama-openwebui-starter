# Contributing

Bedankt voor je bijdrage! Houd wijzigingen klein en gericht, en volg ajb dit proces:

1. Issue aanmaken (of existing issue claimen)
2. Fork/branch maken
3. Code/Docs aanpassen (zie `policy/` voor compliance-sjablonen)
4. `make start` lokaal testen (of `docker compose` varianten)
5. PR openen met ingevulde PR-template

Richtlijnen:
- Geen secrets in de repo. Gebruik `.env` (staat in `.gitignore`).
- Documenteer relevante beslissingen in README/policy.
- Bewaak eenvoud en minimale dependencies.
